----------------------------------------
2.1 VPIN (정보거래자 감지 지표)
----------------------------------------

VPIN (Volume-Synchronized Probability of Informed Trading)

한 줄 설명:
"지금 거래하는 사람들 중에 정보를 가진 사람이 얼마나 있을까?"

쉬운 설명:
  평소: 매수 50% / 매도 50% → 균형 잡힌 시장
  이상: 매수 80% / 매도 20% → 누군가 정보를 알고 매수 중?

VPIN이 높다 = 한쪽으로 쏠린 거래 = 정보거래자가 있을 가능성 높음

실제 예시:
- 09:00 VPIN 0.15 (정상) → 평소처럼 거래
- 09:30 VPIN 0.45 (위험) → 스프레드를 넓히고 물량을 줄여야!
- 09:31 대형 뉴스 발표 → VPIN이 미리 경고해줬음!

활용 방법:
1. VPIN > 0.4 → 스프레드 20% 확대
2. VPIN > 0.5 → 호가 물량 50% 축소
3. VPIN > 0.6 → 일시 거래 중단 검토

학술 배경:
- 2010년 "Flash Crash" 6분 전에 VPIN이 급등
- Easley, López de Prado, O'Hara 교수팀이 개발
- 월가에서 실제로 사용하는 지표

금전적 가치:

Before VPIN:
- 정보거래자에게 당하는 손실: 월 $50,000 추정

After VPIN:
- 70% 사전 감지 가능
- 손실 감소: 월 $35,000
- 연간 절감: $420,000

구현 난이도: 중간 (별 3개)
- 개발 기간: 2-3주
- 필요 데이터: 체결 데이터 (이미 있음)
- 특이사항: 버킷 사이즈 최적화 필요


----------------------------------------
2.2 Kyle's Lambda (시장 영향력 측정)
----------------------------------------

Kyle's Lambda (가격 영향 계수)

한 줄 설명:
"내가 거래하면 가격이 얼마나 움직일까?"

쉬운 설명:
  Lambda가 크다 = 유동성이 적다 = 조금만 거래해도 가격이 확 움직임
  Lambda가 작다 = 유동성이 많다 = 많이 거래해도 가격이 별로 안 움직임

비유:
- 작은 연못에 돌 던지기 (Lambda 큼) → 큰 파도
- 큰 바다에 돌 던지기 (Lambda 작음) → 작은 파도

실제 예시:
  거래량        Lambda = 0.5     Lambda = 2.0
  1 BTC 매수    가격 +0.5 bps    가격 +2.0 bps
  10 BTC 매수   가격 +5 bps      가격 +20 bps
  100 BTC 매수  가격 +50 bps     가격 +200 bps (위험!)

활용 방법:
1. Lambda가 높은 페어 → 스프레드 확대, 물량 축소
2. Lambda가 낮은 페어 → 적극적으로 물량 확대 가능
3. Lambda 급등 시 → 대형 주문 분할 실행 권장

학술 배경:
- Albert Kyle 교수 (1985, Econometrica)
- 노벨상 후보에 오른 금융 이론
- 전세계 헤지펀드에서 사용

금전적 가치:

Before Kyle's Lambda:
- 대형 주문 실행 시 슬리피지: 평균 15 bps
- 월 거래대금 $10M 기준: $15,000 손실

After Kyle's Lambda:
- 최적 실행 전략으로 슬리피지 감소: 8 bps
- 월 절감: $7,000
- 연간 절감: $84,000

구현 난이도: 낮음 (별 2개)
- 개발 기간: 1주
- 필요 데이터: 가격 변화, 거래량 (이미 있음)
- 특이사항: 회귀분석으로 간단히 계산 가능


----------------------------------------
2.3 Amihud Illiquidity (비유동성 측정)
----------------------------------------

Amihud Illiquidity Measure

한 줄 설명:
"이 시장은 얼마나 거래하기 힘든가?"

쉬운 설명:
  Amihud = |가격변화| / 거래대금

  높은 Amihud = 적은 거래대금으로도 가격이 많이 움직임 = 비유동적
  낮은 Amihud = 많은 거래대금에도 가격이 적게 움직임 = 유동적

비유:
  코인 A (Amihud 낮음)     코인 B (Amihud 높음)
  BTC, ETH 같은 대형코인    소형 알트코인
  거래해도 가격 안정        조금만 사도 가격 급등
  마켓메이킹 수월           마켓메이킹 위험

활용 방법:
1. Amihud 높은 페어: 보수적 전략 (넓은 스프레드, 적은 물량)
2. Amihud 낮은 페어: 공격적 전략 (좁은 스프레드, 많은 물량)
3. Amihud 급등 시: 유동성 위기 → 거래 축소

페어별 비교 예시:
  페어           Amihud 지수   추천 전략
  BTC-USDT       0.02          적극 마켓메이킹
  ETH-USDT       0.05          적극 마켓메이킹
  SOL-USDT       0.15          표준 마켓메이킹
  KYO-USDT       0.85          보수적 마켓메이킹
  소형 알트       2.50          마켓메이킹 회피 권장

학술 배경:
- Yakov Amihud 교수 (2002, Journal of Financial Markets)
- 전통 금융에서 수십 년간 검증된 지표

금전적 가치:

Before Amihud:
- 고위험 페어에서 과도한 손실: 월 $30,000

After Amihud:
- 페어별 리스크 조정으로 손실 감소
- 손실 60% 감소: 월 $18,000 절감
- 연간 절감: $216,000

구현 난이도: 낮음 (별 2개)
- 개발 기간: 1주
- 필요 데이터: 수익률, 거래대금 (이미 있음)
- 특이사항: 간단한 나눗셈 계산


----------------------------------------
2.4 Spread Decomposition (스프레드 분해)
----------------------------------------

Spread Decomposition (스프레드 분해 분석)

한 줄 설명:
"우리가 버는 스프레드는 실제로 얼마나 남는 걸까?"

쉬운 설명:
우리가 스프레드 10원을 벌었다고 생각하지만, 실제로는...

  호가 스프레드 (Quoted Spread): 10원
    - 실효 스프레드 (Effective): 8원 (실제 체결 기준)
      - 실현 스프레드 (Realized): 5원 (진짜 남는 수익)
      - 가격 영향 (Price Impact): 3원 (역선택으로 손실)
    - 체결 안 된 부분: 2원

비유:
- 음식점에서 음식 가격이 10,000원이라고 해도
- 재료비 3,000원, 인건비 3,000원, 임대료 2,000원 빼면
- 실제 남는 건 2,000원 (= 실현 스프레드)

왜 중요한가?

  상황 A: 실현 스프레드 > 가격 영향
  → 우리가 돈을 벌고 있다! 전략 유지

  상황 B: 실현 스프레드 < 가격 영향
  → 실제로 손해 보고 있다! 전략 수정 필요
    (정보거래자에게 당하고 있음)

활용 방법:
1. 가격 영향 > 실현 스프레드 → 스프레드 확대 필요
2. 실현 스프레드 >> 가격 영향 → 스프레드 축소 가능 (경쟁력 확보)
3. 시간대별 분해 → 최적 거래 시간대 발견

금전적 가치:

Before Spread Decomposition:
- "10 bps 스프레드니까 잘 벌고 있다"는 착각
- 실제로는 역선택으로 5 bps 손실 중 → 순이익 5 bps

After Spread Decomposition:
- 역선택이 심한 시간대 파악
- 해당 시간대 스프레드 15 bps로 확대
- 순이익 5 bps → 8 bps (60% 수익 증가)
- 연간 추가 수익: 거래대금 대비 3 bps = $300,000+ (월 $10M 기준)

구현 난이도: 높음 (별 4개)
- 개발 기간: 3-4주
- 필요 데이터: 체결 데이터 + 호가 데이터 (시간 동기화 필요)
- 특이사항: 정교한 타임스탬프 매칭 필요

2.5 Implementation Shortfall (실행 부족)
----------------------------------------

Implementation Shortfall (구현 부족 비용)

한 줄 설명:
"결정한 순간부터 실제 체결까지 얼마나 손해봤나?"

쉬운 설명:
  09:00 "1 BTC를 사야겠다" → 가격 $90,000
  09:05 실제 체결 완료 → 평균 체결가 $90,150

  Implementation Shortfall = $150 손실 (0.17%)

손실의 구성:
  총 손실 $150
    - 시장 영향 비용: $80 (내 주문이 가격을 올림)
    - 타이밍 비용: $50 (실행 지연 동안 가격 상승)
    - 기회 비용: $10 (일부 미체결로 상승분 놓침)
    - 수수료: $10

왜 중요한가?
- 우리 전략의 "숨겨진 비용"을 정확히 알 수 있음
- 어디서 손실이 발생하는지 파악 → 개선 가능
- 알고리즘 트레이딩 성과 평가의 표준 지표

활용 방법:
1. 시장 영향 비용이 크다 → 주문 분할 실행
2. 타이밍 비용이 크다 → 실행 속도 개선
3. 기회 비용이 크다 → 체결률 개선 필요

금전적 가치:

Before IS Analysis:
- 실행 비용 인지 못함
- 평균 IS: 15 bps

After IS Analysis:
- 비용 구조 파악 후 최적화
- IS 10 bps로 감소 (33% 개선)
- 월 거래대금 $10M 기준: $5,000/월 절감
- 연간 절감: $60,000

구현 난이도: 중간 (별 3개)
- 개발 기간: 2주
- 필요 데이터: 주문 생성 시점 + 체결 데이터
- 특이사항: 결정 시점 정의가 중요

3.1 DVR (Depth-to-Volume Ratio) - 유동성 안정성 지표
----------------------------------------

DVR (Depth-to-Volume Ratio)

한 줄 설명:
"지금 쌓여있는 호가로 현재 거래량을 감당할 수 있을까?"

쉬운 설명:
  DVR = (매수 호가 물량 + 매도 호가 물량) / 분당 거래량

  DVR > 1.0 → 호가 물량이 거래량보다 많음 → 안정적
  DVR < 0.5 → 호가 물량이 부족함 → 가격 변동 위험
  DVR < 0.2 → 유동성 위기! → 즉시 대응 필요

비유 (마트):
  DVR 높음: 창고에 재고 충분, 손님 많이 와도 OK
  DVR 낮음: 재고 부족, 손님 몰리면 품절 위기!

실제 예시:
  시간      호가 물량   분당 거래량   DVR (상태)
  09:00     5 BTC       3 BTC        1.67 (안정)
  09:15     4 BTC       6 BTC        0.67 (주의)
  09:19     2 BTC       42 BTC       0.05 (위험! 스파이크)
  09:20     3 BTC       4 BTC        0.75 (회복 중)

활용 방법:
1. DVR < 0.5 → 스프레드 확대 (최소 20%)
2. DVR < 0.2 → 호가 물량 축소 또는 일시 철회
3. DVR 급락 → 대형 거래 경고 발송

금전적 가치:

Before DVR:
- 유동성 위기 시 대응 지연
- 불리한 가격에 체결: 월 $20,000 손실

After DVR:
- 실시간 유동성 모니터링
- 80% 사전 대응 가능
- 손실 감소: 월 $16,000
- 연간 절감: $192,000

구현 난이도: 매우 낮음 (별 1개)
- 개발 기간: 3일
- 필요 데이터: 호가 깊이, 거래량 (이미 있음)
- 특이사항: 단순 나눗셈, 즉시 구현 가능

3.2 TII (Trade Intensity Index) - 거래 강도 지표
----------------------------------------

TII (Trade Intensity Index)

한 줄 설명:
"가격이 조금 움직이는데 거래는 얼마나 많이 일어났나?"

쉬운 설명:
  TII = 거래 건수 / |가격 변화(bps)|

  TII 높음 → 많이 거래해도 가격이 별로 안 움직임 → 유동성 좋음
  TII 낮음 → 조금만 거래해도 가격이 크게 움직임 → 유동성 나쁨

비유 (도로):
  TII 높음: 8차선 고속도로 - 차 많아도 잘 흐름
  TII 낮음: 1차선 골목길 - 차 몇 대만 막혀도 정체

활용 방법:
1. TII 급락 → 시장 스트레스 신호, 주의 필요
2. TII 지속 높음 → 대형 주문 실행에 좋은 타이밍
3. 시간대별 TII 분석 → 최적 거래 시간대 발견

금전적 가치:

활용 효과:
- 최적 시간대 거래 집중
- 슬리피지 감소: 평균 3 bps
- 연간 절감: $36,000 (월 $10M 거래 기준)

구현 난이도: 매우 낮음 (별 1개)
- 개발 기간: 2일
- 필요 데이터: 거래 건수, 가격 변화 (이미 있음)


----------------------------------------
3.3 FPI (Flow Persistence Indicator) - 흐름 지속성 지표
----------------------------------------

FPI (Flow Persistence Indicator)

한 줄 설명:
"지금 매수/매도 흐름이 계속될 것인가, 곧 바뀔 것인가?"

쉬운 설명:
  FPI = (연속 같은 방향 분 수 x 누적 순거래량) / 총 거래량

  FPI > 0.3 → 강한 추세 (매수 또는 매도가 계속됨)
  FPI < 0.1 → 방향 없음 (횡보, 마켓메이킹에 유리)

비유 (파도):
  FPI 높음: 큰 파도 - 방향성 강함, 맞서면 위험
  FPI 낮음: 잔잔한 물결 - 방향성 약함, 양쪽 호가 안전

활용 방법:
  FPI 높음 (추세장)              FPI 낮음 (횡보장)
  - 추세 방향으로 편향 호가        - 양쪽 균등 호가
  - 스프레드 축소 가능            - 스프레드 유지
  - 역방향 물량 축소              - 물량 확대 가능

금전적 가치:

활용 효과:
- 추세장/횡보장 구분으로 전략 최적화
- 역방향 손실 감소: 월 $8,000
- 연간 절감: $96,000

구현 난이도: 낮음 (별 2개)
- 개발 기간: 1주
- 필요 데이터: Buy Ratio 시계열 (이미 있음)


----------------------------------------
3.4 WAS (Whale Activity Score) - 고래 활동 점수
----------------------------------------

WAS (Whale Activity Score)

한 줄 설명:
"지금 큰손(고래)이 시장에서 활동하고 있나?"

쉬운 설명:
  WAS = 대형거래 비중 x 0.5 + 집중도 x 0.3 + 타이밍 패턴 x 0.2

  WAS > 0.7 → 고래 활동 감지! 주의 필요
  WAS < 0.4 → 소매 거래 위주, 정상 운영

왜 중요한가?
  고래가 움직이면...
  - 가격이 크게 변동할 수 있음
  - 우리 호가가 한번에 소진될 수 있음
  - 정보거래 가능성 높음
  - 따라서 미리 대비해야 손실 최소화!

실제 예시 (2026-01-23 데이터):
  시간      상태
  09:18     WAS 0.35 (정상)
  09:19     WAS 0.92 (고래 감지!)
            → 42 BTC 대형 매도 발생
            → Buy Ratio 5.1% (극단적 매도)
  09:20     WAS 0.65 (아직 주의)

활용 방법:
1. WAS > 0.7 → 경고 알림 발송
2. WAS > 0.8 → 스프레드 30% 확대

3.5 LSI (Liquidity Stress Index) - 유동성 스트레스 지수
----------------------------------------

LSI (Liquidity Stress Index)

한 줄 설명:
"지금 시장의 유동성 상태가 얼마나 위험한가?"

쉬운 설명:
  LSI = 스프레드 이탈 x 0.3 + 깊이 감소 x 0.3
      + 거래 긴급도 x 0.2 + 변동성 x 0.2

  LSI < 1.0 → 정상 상태, 평소처럼 운영
  LSI 1.0~1.5 → 경미한 스트레스, 주의 필요
  LSI 1.5~2.0 → 중간 스트레스, 스프레드 확대
  LSI > 2.0 → 심각한 스트레스, 거래 축소 또는 중단 검토

비유 (체온계):
  LSI < 1.0: 체온 36.5도C (정상)
  LSI 1.0~1.5: 체온 37.5도C (미열, 관찰 필요)
  LSI 1.5~2.0: 체온 38.5도C (발열, 조치 필요)
  LSI > 2.0: 체온 40도C (고열, 응급 상황!)

각 구성요소 설명:
  구성요소          의미
  스프레드 이탈     평소보다 스프레드가 얼마나 넓어졌나
  깊이 감소        평소보다 호가 물량이 얼마나 줄었나
  거래 긴급도      Taker 비율이 평소보다 얼마나 높은가
  변동성          평소보다 가격이 얼마나 많이 움직이나

활용 방법 (LSI 레벨별 대응 프로토콜):
  LEVEL 1 (LSI < 1.0): 정상 운영
  LEVEL 2 (LSI 1.0~1.5): 모니터링 강화, 알림 대기
  LEVEL 3 (LSI 1.5~2.0): 스프레드 +30%, 물량 -30%
  LEVEL 4 (LSI > 2.0): 스프레드 +50%, 물량 -50%, 경고 발송
  LEVEL 5 (LSI > 3.0): 거래 일시 중단, 수동 검토 필요

금전적 가치:

Before LSI:
- 유동성 위기 시 무방비 손실
- 위기 시 평균 손실: 월 $35,000

After LSI:
- 단계적 대응으로 손실 최소화
- 85% 손실 방어
- 손실 감소: 월 $29,750
- 연간 절감: $357,000

구현 난이도: 중간 (별 3개)
- 개발 기간: 2주
- 필요 데이터: 스프레드, 깊이, 거래 비율, 변동성 (이미 있음)
- 특이사항: 가중치 최적화 필요
